# 第二阶段端到端演示脚本

## 1. 演示目标

验证第二阶段核心业务闭环：

```text
登录
-> 查看资源档案
-> 提交合作申请
-> 查看我的申请
-> 查看流程详情和操作日志
-> 审批处理
-> 报表查看申请与审批指标
```

## 2. 前置条件

基础设施已启动：

```powershell
docker compose -f docker-compose.demo.yml up -d
```

后端服务已启动：

```powershell
cd backend
.\scripts\start-auth.ps1
.\scripts\start-operation.ps1
.\scripts\start-analysis.ps1
.\scripts\start-gateway.ps1
```

小程序配置：

```text
dataSource = api
baseURL = http://127.0.0.1:8080/api
```

## 3. 演示账号

| 账号 | 角色 | 用途 |
| --- | --- | --- |
| `user_demo` | USER | 提交合作申请、补充材料 |
| `staff_demo` | STAFF | 审批申请、要求补充材料 |
| `admin` | ADMIN | 查看全局管理和刷新看板 |

## 4. 演示步骤

### 4.1 用户提交申请

1. 使用 `user_demo` 登录小程序。
2. 进入资源地图或资源详情。
3. 打开任一 `可招商` 资源。
4. 确认资源档案展示：
   - 权属状态
   - 材料状态
   - 招商说明
   - 现场图片提示
5. 点击“提交合作申请”。
6. 填写合作说明和联系方式。
7. 提交后进入“我的申请”。

验收：

- 申请状态为 `待审批`。
- “我的申请”中能看到申请编号、资源编号、提交时间。

### 4.2 查看流程详情

1. 在“我的申请”中点击申请卡片。
2. 进入流程详情页。
3. 查看审批记录和操作日志。

验收：

- 操作日志中出现“提交申请”。
- 日志展示操作人、操作时间和备注。

### 4.3 工作人员审批

1. 使用 `staff_demo` 登录。
2. 进入协同工作台或流程详情。
3. 对申请执行审批通过或驳回。

验收：

- 申请状态变为 `已通过` 或 `已驳回`。
- 审批记录展示处理意见。
- 操作日志中出现“审批处理”。

### 4.4 补充材料流程

1. 工作人员对申请执行 `MATERIAL_REQUIRED` 动作。
2. 用户再次打开流程详情。
3. 页面展示补充材料输入框。
4. 用户提交补充说明。

验收：

- 申请状态回到 `待审批`。
- 操作日志中出现“补充材料”。

### 4.5 报表联动

1. 打开智能报表页。
2. 查看摘要指标。

验收：

- 摘要中展示合作申请数。
- 摘要中展示审批通过率。
- 摘要中展示超时待办数。
