# V1.3 Admin Pro：管理员工作台生产化增强

V1.3 的目标是把管理端从“能演示、能操作”推进到“能管理、能追责、能长期使用”的单村受控试点平台。

本阶段只完成应用层管理员能力生产化，不包含高可用、灾备、Kubernetes、灰度发布、ELK 或完整安全运营平台。

## 发布节点

### `v1.3-admin-pro-rc1`

安全与数据可信基线：

- 后端角色硬约束，越权请求返回 HTTP 403。
- 管理员看板使用按村域统计的真实数据。
- 统一业务异常的 HTTP 状态与业务错误码。
- 建立统一分页协议和不可随业务接口删除的审计日志底座。
- 补齐权限、统计、分页和审计测试。

### `v1.3-admin-pro`

长期使用能力：

- 审计中心页面和操作详情。
- 资源材料上传、预览、下载、封面与软删除。
- 服务端全局搜索和安全的批量操作。
- 系统设置、个人中心、登录记录和持久化通知。
- 最小 Playwright 管理员主流程回归。

## RC1 当前进度

截至 2026-07-15，`feature/v1.3-admin-pro` 已完成第一批：

- [x] USER 不能新增、编辑或删除资源，不能生成周报，不能执行审批。
- [x] STAFF 可以维护资源、处理审批和周报，但不能发布、下架或删除资源。
- [x] ADMIN 保留用户、角色、村域、资源生命周期和系统级管理权限。
- [x] 补材料接口校验流程申请人，不能为他人的流程补材料。
- [x] `BusinessException` 按错误码返回 400/401/403/404/409 等 HTTP 状态。
- [x] `GET /api/dashboard/admin-overview` 与兼容入口 `GET /api/dashboard` 返回真实管理员概览。
- [x] 资源总数、可合作资源、待审批、超时流程和资源分类均按 `village_id` 实时计算。
- [x] 前端已移除固定资源占比、假“较昨日”变化和二次拼接待办数。
- [x] 运行时验证：资源 17、可合作 10、待审批 3；USER 写资源和 STAFF 发布资源均返回 403。
- [x] Web 生产构建、权限测试、业务服务测试和分析服务测试通过。

## 权限边界

| 能力 | USER | STAFF | ADMIN |
| --- | --- | --- | --- |
| 查看公开业务数据、提交自己的合作申请 | 是 | 是 | 是 |
| 为自己的流程补材料 | 是 | 是 | 是 |
| 资源新增与编辑 | 否 | 是 | 是 |
| 资源发布、下架、删除 | 否 | 否 | 是 |
| 审批、驳回、要求补材料 | 否 | 是 | 是 |
| 周报生成与维护 | 否 | 是 | 是 |
| 用户、角色、村域和系统级配置 | 否 | 否 | 是 |

前端隐藏或禁用按钮只负责体验，Gateway 和业务服务负责最终鉴权与对象归属校验。

## 后续执行顺序

1. 统一分页响应：`items`、`page`、`pageSize`、`total`、`totalPages`。
2. 审计底座：成功、失败、越权、Trace ID、操作对象和前后差异。
3. 审计中心页面：筛选、搜索、详情和 CSV 导出。
4. 资源材料管理：鉴权下载、随机存储名、软删除、上传失败回滚。
5. 服务端全局搜索、危险操作二次确认和批量结果汇总。
6. 设置中心、个人中心、通知持久化和 E2E 验收。

正式文件不得写入 Git、`target`、`src/main/resources` 或 `web-admin/public`。本地存储实现应使用部署目录（例如 `/data/xiangyun/uploads`），并通过鉴权接口访问。
