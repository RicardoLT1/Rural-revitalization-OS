# V1.3 Admin Pro：管理员工作台生产化增强

V1.3 的目标是把管理端从“能演示、能操作”推进到“能管理、能追责、能长期使用”的单村受控试点平台。

本阶段只完成应用层管理员能力生产化，不包含高可用、灾备、Kubernetes、灰度发布、ELK 或完整安全运营平台。

## 发布节点

### `v1.3-admin-pro-rc1`

安全与数据可信基线：

- 后端角色硬约束，越权请求返回 HTTP 403。
- 管理员看板使用按村域统计的真实数据。
- 统一业务异常的 HTTP 状态与业务错误码。
- 建立统一分页协议和不可随业务接口删除的审计日志底座。
- 补齐权限、统计、分页和审计测试。

### `v1.3-admin-pro`

长期使用能力：

- 审计中心高级筛选、变更前后差异与导出。
- 资源材料上传、预览、下载、封面与软删除。
- 服务端全局搜索和安全的批量操作。
- 系统设置、个人中心、登录记录和持久化通知。
- 最小 Playwright 管理员主流程回归。

## RC1 当前进度

截至 2026-07-15，`feature/v1.3-admin-pro` 已完成 RC1 前两批：

- [x] USER 不能新增、编辑或删除资源，不能生成周报，不能执行审批。
- [x] STAFF 可以维护资源、处理审批和周报，但不能发布、下架或删除资源。
- [x] ADMIN 保留用户、角色、村域、资源生命周期和系统级管理权限。
- [x] 补材料接口校验流程申请人，不能为他人的流程补材料。
- [x] `BusinessException` 按错误码返回 400/401/403/404/409 等 HTTP 状态。
- [x] `GET /api/dashboard/admin-overview` 与兼容入口 `GET /api/dashboard` 返回真实管理员概览。
- [x] 资源总数、可合作资源、待审批、超时流程和资源分类均按 `village_id` 实时计算。
- [x] 前端已移除固定资源占比、假“较昨日”变化和二次拼接待办数。
- [x] 运行时验证：资源 17、可合作 10、待审批 3；USER 写资源和 STAFF 发布资源均返回 403。
- [x] Web 生产构建、权限测试、业务服务测试和分析服务测试通过。
- [x] 资源、审批、用户和周报列表统一返回 `items`、`page`、`pageSize`、`total`、`totalPages`。
- [x] 资源、审批和用户管理页使用服务端分页，筛选、翻页与真实总数保持一致。
- [x] 新增 `admin_audit_log` 审计表，普通业务接口不提供删除能力。
- [x] 资源、审批、待办和周报的写操作记录操作人、角色、村域、IP、设备、Trace ID、结果与耗时。
- [x] 管理端新增仅 ADMIN 可见的“审计日志”页面，支持关键词、模块、结果筛选和证据详情。
- [x] `GET /api/dashboard/admin-overview` 仅 STAFF / ADMIN 可访问，普通用户仍使用兼容看板入口。

### 第三批：审计闭环增强

- [x] 新增带内部签名的跨服务审计事件入口，认证服务和网关可以统一写入运营服务审计库。
- [x] 网关拒绝 USER / STAFF 越权请求时持久化 `ACCESS_DENIED`，保留操作人、角色、路径、HTTP 403 和 Trace ID。
- [x] 用户新增、编辑、启停、重置密码、角色维护和角色分配进入统一审计链路。
- [x] 资源创建、编辑、删除、发布、下架和招商状态变更记录操作前后快照。
- [x] 审计详情按字段对比变更前后值，只突出真正变化的证据；资源、用户、安全模块可独立筛选。
- [x] 密码重置只记录“密码已变更、会话已失效”，不记录明文密码或密码摘要。
- [x] 登录成功、登录失败和退出登录统一进入 SECURITY 审计，失败登录不保存密码。
- [x] 审计中心支持开始/结束日期筛选，关键词可联查操作人、对象与 Trace ID。
- [x] 当前筛选结果可导出带 UTF-8 BOM 的 CSV，并对表格公式注入字符做转义处理。
- [x] 跨服务审计失败不会反向阻断已经成功的用户管理或网关鉴权响应。
- [x] 运行态回归通过：未签名直连认证服务返回 403；USER 写资源约 1 秒内返回 403 并生成安全审计；用户和资源变更均生成前后快照。

第三批完成后，RC1 已具备应用层关键管理操作、登录安全事件、越权访问和字段变更的集中追踪与导出能力。外部不可篡改归档、集中安全告警和长期归档策略属于生产运维层后续增强，不在当前应用层 Admin Pro 范围内。

### 第四批：资源材料管理

- [x] 新增 `file_object` 与 `resource_material` 正式材料模型，业务元数据与物理文件分离，删除采用软删除。
- [x] 本地存储默认使用部署目录 `/data/xiangyun/uploads`，文件使用日期分区和随机存储名，不写入 Git、`target`、资源目录或 Web 公共目录。
- [x] 上传限制为 10 MB，校验扩展名、MIME、图片/PDF/Office 文件头以及 DOCX/XLSX 包结构，并计算 SHA-256 摘要。
- [x] 上传事务失败会清理物理文件；文件名经过路径清理，存储键经过根目录越界校验。
- [x] STAFF / ADMIN 可归档材料、维护标题和说明；替换、删除和设置封面仅 ADMIN 可执行。
- [x] 现场照片分类只接受图片，只有现场照片分类的图片可设为资源封面；删除当前封面时自动提升下一张现场照片。
- [x] 预览和下载必须经过带身份认证的接口，不暴露实际存储路径；响应包含 `nosniff`、沙箱策略和私有缓存策略。
- [x] 资源详情新增“材料档案带 + 封面卡”，支持分类统计、上传、鉴权预览、下载、说明维护、替换、封面和危险删除确认。
- [x] 旧版现场照片保留为明确标识的只读历史资料，避免迁移期间出现资料不可见。
- [x] 上传、说明更新、替换、设置封面和删除均接入管理员审计中心，成功与越权失败均可追踪。
- [x] 全量后端 103 个测试与 Web 生产构建通过；运行态验证 USER 读取 403、STAFF 上传 200 / 删除 403、ADMIN 封面/内容/删除 200，软删除后列表不可见。

第四批完成后，资源详情已从“现场照片展示”升级为具备文件安全、角色边界、审计留痕和生命周期管理的受控材料档案。对象存储、病毒扫描、保留期清理与外部归档属于正式生产部署增强，不在当前本地试点实现范围内。

### 第五批：服务端全局搜索与安全批量操作

- [x] 新增 `GET /api/search` 统一搜索协议，按当前 `village_id` 联查资源、流程与周报，ADMIN 可额外联查当前村域用户。
- [x] 搜索关键词限制为 50 个字符、单次最多返回 20 条；USER 访问返回 403，STAFF 不会获得用户目录结果。
- [x] 用户联查通过带内部签名的服务间接口完成；认证服务短暂失败时返回核心业务的部分结果，不让全局搜索整体不可用。
- [x] 顶栏搜索移除资源、待办、周报和用户的前端全量预加载，改为 220ms 防抖服务端查询，并处理并发结果覆盖、加载、错误、空结果和最近访问。
- [x] 搜索结果可直接下钻资源档案、审批流程、周报管理和带关键词的用户管理页面，Enter 可打开首条结果，Esc 可关闭。
- [x] 新增 `POST /api/resources/batch/actions`；仅 ADMIN 可批量发布或下架，单次最多 50 个去重资源编号。
- [x] 危险操作确认明确说明影响；后端逐条执行并返回请求、成功、失败和逐项原因，单条失败不会掩盖已成功结果。
- [x] 批量操作记录目标资源、操作类型和汇总结果到审计中心；STAFF 调用由网关返回 403 并形成越权记录。
- [x] Web 生产构建和相关后端测试通过；运行态验证 STAFF 搜索 5 条且无用户泄露、ADMIN 联查 2 个用户、USER 搜索 403、批量操作 1 成功 / 1 失败且审计可查。

第五批完成后，管理员不再依赖浏览器预加载的小数据索引，并具备可控、可解释、可审计的资源批处理能力。

## 权限边界

| 能力 | USER | STAFF | ADMIN |
| --- | --- | --- | --- |
| 查看公开业务数据、提交自己的合作申请 | 是 | 是 | 是 |
| 为自己的流程补材料 | 是 | 是 | 是 |
| 资源新增与编辑 | 否 | 是 | 是 |
| 资源发布、下架、删除 | 否 | 否 | 是 |
| 审批、驳回、要求补材料 | 否 | 是 | 是 |
| 周报生成与维护 | 否 | 是 | 是 |
| 用户、角色、村域和系统级配置 | 否 | 否 | 是 |

前端隐藏或禁用按钮只负责体验，Gateway 和业务服务负责最终鉴权与对象归属校验。
